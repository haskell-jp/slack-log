何かを習得するとき一般に言えることですが、複雑なことを成し遂げたいときはまず個々の動きから始めましょう。この場合の考え方は
• QuickCheckは純粋な関数をテストする仕組みである
• 簡単なもの(足し算の交換則など)をQuickCheckしてみて、基本的な使い方を知る
• SFを純粋に検査できるようにする(embed)
• 両者を組み合わせ、目標を達成する
