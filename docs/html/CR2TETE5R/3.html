ghciを起動すると、`Prelude>` と表示されます。これはどういう意味・意図ですか？ [連載QA]
takenobu.hs
2019-12-28 16:18:30 +0900
これは、ghciが現在読み込んでいるモジュール（importしているモジュール）を表しています。
つまり現在、"Prelude"モジュールが、importされていることを示しています。
ghciは起動時に暗黙的・自動的に、"Prelude"モジュールをimportするため、それが表示されています。

例えば、次のように、"Data.List"モジュールを追加でimportすると、それらがプロンプトに追加されます。
Prelude> import Data.List

Prelude Data.List>

なお、次のように、`set prompt`コマンドにより、プロンプトを変更することが出来ます。
Prelude> :set prompt "ghci> "

ghci>

また、`~/.ghci` ファイルを作り、次のように設定を書いておくと、ghciのデフォルトのプロンプトを変えることが出来ます。
:set prompt "ghci> "

「絶対に古いGHCをインストールしたくない！」というのはstackの設計思想上難しいです。
stackは、パッケージのバージョンとGHCのバージョンを組み合わせて「resolver」としてまとめて固定することで確実にビルドすることを保証する、というソフトウェアなので。

単純にもう使わないであろうGHCを消したい、という場合は、単純に対象のGHCがインストールされているディレクトリーを探して削除しましょう。

手元の環境はWindowsなのでMacやLinuxの場合は自信がないのですが、
~/.stack 以下のどこかに ghc-8.6.5 などといったディレクトリーがあるはずです。

ただし、そうして削除した後でも、もう一度対象のGHCに依存したパッケージのディレクトリーで stack build を実行すると、再度インストールすることになってしまうので「このバージョンのGHCに依存するパッケージはもうビルドしない！」と確信が持てているときだけ削除してください。
